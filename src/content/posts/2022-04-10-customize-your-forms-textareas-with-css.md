Nowadays when you look at different web sites and what makes them successful, you see different fonts and effortless looking styling and amazing accessibility, but there is another thing that can be extremely important and helpful to your site visitors—forms. These forms can be found in a variety of different ways, whether it’s a contact form, a search form or even data filtering. There are so many different possibilities for forms to be integrated into your webpage, but a crucial part of creating a form is making it readable and accessible which can be achieved using CSS to style your form to perfection. While it is impossible to say what perfection is, the goal of this article is to help you take your first steps towards creating the form you would be proud to have on your webpage. Throughout this article you will learn about basic structure of a form, some simple and good habits when creating CSS for your forms, a few unique inputs/elements you can add into your form, and how to utilize CSS to format your forms how you like.

To begin, we will start with the basic structure of a form. For the sake of consistency, we will we using a Contact Us Form as our example throughout this article. The format we will be using is a label & input for Name, a label & input for Email, a label & dropdown for whether you are entering a Home, Mobile, Work, or No Phone, an input for Phone Number, 2 radio options for whether you would want a response back or not, a place for you to type out a message, and finally both a submit button and a clear form button. While there are more elements you can add to your form, we will be using this format to show a few common choices and different ways you can customize them. Below is the base html for the form listed above:

![](/assets/html-1.png)

The next step after writing out your html for you form is creating some basic CSS styling for it. For example, by using the following CSS you can make your form have some shape to it (using some basic formatting to the page as well), instead of it being a jumbled up mess of items in a corner of your page:

![](/assets/css-1.png)

With all of that HTML and CSS, we have now finished making the base structure of a form, so we can now get to the fun part of forms—customizing.

There are tons of ways that you can customize your form to your liking, but because we are not insane and don’t plan on writing a book about it, we will start with some basics and dive a little more into some specific options. The very first thing you should start with is removing the default browser styling for the button, fieldset, input, legend, select, and textarea. You can do this by creating a selector group with those elements and adding the appearance property and setting it to none. This will remove those default browser styling from those elements so there is more consistency across different browsers. At the time of writing this most browsers still require a vendor prefix for this property, so make sure to double check all of them to make sure everything works as intended later down the line.

![](/assets/vendor-prefix.png)

After that you can start formatting the individual or small groups of elements to your liking. Two things that you can do is add specific borders (sizing, color, radius) and background colors to each input if you wanted to, but it is best if you keep that at least somewhat consistent across all the elements of your form. For example, if you set the input, select, and textarea to the following code, you can have them appear with a nice round and slightly thick dark border around the input boxes:

![](/assets/border.png)

Of course, you should also make sure to adjust paddings, margins, font size & weight, and all those minor things as you go because each time you make a change the styling of the boxes or other input fields, you could drastically affect the output of your code easily. Now you may have noticed that when we cleared the default browser styling for the form elements, the radio options for whether you want a response back or not don’t look right, and that’s okay we are going to fix that now. By manually setting the height & width as well as the border radius of the input type radio and by adding a “background-image” with a gradient to the radio option that is checked, we are able to create the radio options and have control over customizing them without worrying about the browser defaulting it back to their settings.

![](/assets/full-customize.png)

Most of the rest for labels and input customization that we talked about it just personal preferences on font sizing, margins, and all that jazz. But before we move on there are two small things to note, one being if you want a placeholder inside your inputs like saying “First or Full Name” for the Name field, you need to put inside the html for your input: <input id=”name” type=”text” placeholder=”First or Full Name” />. The other being you can customize your submit and clear form buttons by using the selectors: button\[type=”submit”] and button\[type=”reset”] and then adding your CSS for them like normal.

The last thing I want to talk about is formatting textareas specifically. Textareas have multiple different ways you can style them, ranging from if and how you can resize the textarea, to whether or not you can enter text into them, and even having specific CSS styling when you hover or are “focused” on an input. Let’s start with how to specify if and how you want textareas to be resizable. Usually when creating a contact form, you want there to be an option to send a message, but sometimes you don’t want it to be resizable or you want it to only be changeable in 1 direction, this can be done with the resize property. For the most part, the different options you set it to include both (making it resizable to both vertical and horizontal directions), horizontal (for ONLY horizontal), vertical (for ONLY vertical), and none for if you don’t want it to be resizable at all. And that’s it for the resize property, it’s pretty straightforward. The next thing you can do for can do for textareas (as well as other aspects on your page) is the hover and focus state styles. Yes, you can apply both to your textareas as well. This opens up the possibilities of having the background change slightly or the border become blue to signify that you are currently entering into or hovering over the textarea. This is done by doing: textarea:focus or textarea:hover and then applying your styles. You can also use the \[readonly] attribute in CSS to make the entry field well, read-only instead of someone being able to enter in it. If you don’t want it to be read-only, but you also don’t want people to be able to submit anything for it, you can set the attribute to \[disabled] instead and use CSS to update it to show that it is disabled, maybe by putting a muted grey as the background and border.
